# KASUMI Intelligent Workspace Platform

**KASUMI** is an AI-native, enterprise-grade desktop workspace platform comprising two integrated shells:

- **KASUMI Nexcel** ‚Äî Excel-like spreadsheet shell over a structured database backend
- **KASUMI WORDO** ‚Äî Word-like document editor with AI-first document semantics

> The frontend looks like Office. The backend stays structured and AI-native. Both shells share a unified platform layer so data flows freely between them.

## Vision

KASUMI eliminates the friction between **database-thinking** and **document-acting**. Users interact with data using familiar Office patterns ‚Äî keyboard shortcuts, range selections, rich-text editing, page layout ‚Äî while the system validates every write against a typed schema and AI agents operate against stable, structured data contracts.

**Non-goals (by design):**
- VBA / macros
- Full formula engine
- Pivot tables
- Freeform computation that bypasses data contracts

These are excluded because AI agents replace the advanced analytical and authoring layers, requiring deterministic operations and stable data semantics.

## KASUMI Nexcel

Enterprise-grade Excel shell over [Baserow](https://baserow.io), delivering familiar spreadsheet interaction while preserving structured, AI-native data semantics.

### Grid Engine

| Feature | Details |
|---------|---------|
| Virtualized rendering | `@tanstack/react-virtual` for both axes ‚Äî handles 10,000+ rows without DOM explosion |
| Sticky headers | Fixed header bar synced to scroll container via DOM ref |
| Column resize | Drag handle on header right edge |
| Auto-fit column | Double-click or right-click ‚Üí Auto-fit Width |
| Frozen columns | CSS `position: sticky` with computed left offsets |
| Hidden columns | Toggle via right-click header menu |
| Field type rendering | Registry-based dispatcher ‚Äî text, number, date, boolean, single/multi-select, URL, email, phone, formula |

### Selection & Editing

| Feature | Details |
|---------|---------|
| Range selection | Anchor + focus model; `Shift+Click`, `Shift+Arrow`, click-drag |
| Inline editors | Text, number, date, single-select portal, multi-select portal, boolean toggle |
| Formula bar | Bound to store `editValue`; shows `fieldName[rowIndex]` reference |
| Commit pipeline | Optimistic patch ‚Üí adapter call ‚Üí rollback on error |
| Field lock | Formula/computed fields blocked from writes (üîí icon) |

### Clipboard, Fill & History

| Feature | Details |
|---------|---------|
| Copy / Paste / Cut | TSV-compatible ‚Äî pasteable into Excel/Sheets |
| Fill handle | 6√ó6 green square; drag to fill or increment numeric/date series |
| Undo / Redo | 50-step client-side stack (`Ctrl+Z` / `Ctrl+Y`) |

### Import / Export

| CSV | ‚úÖ Header-matched column mapping | ‚úÖ Browser download |

## KASUMI WORDO

Word-like document editor built on ProseMirror with a Section-per-instance architecture and first-class Nexcel data embedding.

### Document Model

WORDO uses a canonical **Kasumi Document IR**:

```
KasumiDocument
  ‚îî‚îÄ DocumentSection[]       ‚Üê each section = one ProseMirror instance
       ‚îî‚îÄ AnyBlock[]         ‚Üê paragraph, heading, list, table, quote, code, nexcel_embed
```

Each `DocumentSection` is an independent ProseMirror instance managed by the **Layout Orchestrator**. This provides:
- Performance isolation (edits on section 1 don't reflow section 50)
- Precise AI targeting via `section_id`
- Natural section break semantics (different headers/footers, page orientation)

### Editor Features

| Feature | Details |
|---------|---------|
| Rich text blocks | Paragraph, Heading H1‚ÄìH6, Bullet list, Ordered list, Blockquote, Code block |
| Table | ProseMirror native table with cell selection |
| Section breaks | Insert section break ‚Üí new ProseMirror instance with independent settings |
| Outline panel | Live H1‚ÄìH6 heading tree; click to jump to section |
| Header / Footer | Independent ProseMirror instances per section; supports page number tokens |
| Watermark | Text, opacity, angle ‚Äî absolutely positioned overlay |
| Page settings | A4 / A3 / Letter / Legal; portrait / landscape; margin control (mm) |

### Nexcel Data Embedding

WORDO documents can embed live Nexcel data in two modes:

| Mode | Behaviour |
|------|-----------|
| **Snapshot** | Point-in-time copy of Nexcel table data; static in document |
| **Live Link** | Subscribes to Nexcel store; renders current data on each view |

Embedded tables are rendered as `NexcelEmbedView` ProseMirror NodeViews and appear inline in the document flow.

### Import / Export

| Format | Direction | Notes |
|--------|-----------|-------|
| `.docx` | Export | `docx` library ‚Äî all block types mapped, includes header/footer |
| `.docx` | Import | `mammoth` ‚Üí HTML ‚Üí ProseMirror; multi-section support |
| `.pdf` | Export | Browser print dialog with `@media print` pagination |

### High-Fidelity .docx Pipeline

WORDO includes a full high-fidelity document layer for real-world `.docx` files:

| Service | Purpose |
|---------|---------|
| `DocxImporter` | Converts `.docx` ‚Üí canonical Kasumi Document IR (images, spacing, fonts, section layout) |
| `DocxExporter` | Exports document IR ‚Üí `.docx` with full formatting round-trip |
| `DocumentFidelity` | Scores import quality across 10+ fidelity dimensions (fonts, images, tables, spacing‚Ä¶) |
| `FidelityRegression` | Detects fidelity regressions across importer versions |
| `PaginationSnapshotBuilder` | Builds paginated layout snapshots for high-fidelity page-accurate preview |
| `DocxPreviewSurface` | Renders paginated `.docx` preview with Word-accurate page boundaries |
| `DocxDiagnosticsBanner` | Shows import warnings and fidelity score inline in the editor |

**Design principle:** `Render fidelity for humans, semantic fidelity for AI.`

See [docs/WORDO_HIGH_FIDELITY_AI_NATIVE_UPGRADE_PLAN.md](docs/WORDO_HIGH_FIDELITY_AI_NATIVE_UPGRADE_PLAN.md) for the full architecture.

### MCP Adapter

WORDO exposes a native MCP tool surface via `WordoMcpAdapter` and `SemanticCommandService`:

| Component | Role |
|-----------|------|
| `SemanticCommandService` | Unified execution pipeline for all AI document commands |
| `WordoMcpAdapter` | MCP tool delegation ‚Äî wraps semantic commands as MCP-callable tools |
| `CommandAuditExporter` | Records all AI commands with timestamps for audit / replay |

This enables AI agents (e.g. Minato via MCP) to read, reason about, and safely modify document structure without DOM access.

### AI Command System

WORDO exposes 17 typed commands through the platform command-bus:

| Command | Description |
|---------|-------------|
| `InsertBlock` | Insert any block type at a given position |
| `DeleteBlock` | Remove a block by ID |
| `RewriteBlock` | Replace block content via AI-generated text |
| `InsertNexcelEmbed` | Embed a Nexcel table (snapshot or live) |
| `InsertSectionBreak` | Add a new document section |
| `UpdatePageSettings` | Change paper size / orientation / margins |
| `SetWatermark` | Apply or remove watermark |
| `UpdateHeaderFooter` | Modify header/footer content |
| + 9 more | See `platform/command-bus` for full list |

All AI commands operate through validated typed operations ‚Äî they never touch the DOM directly.

## Getting Started

### Prerequisites

- **Node.js** 18 LTS or later
- **npm** 9+
- A running **Baserow** instance (optional ‚Äî mock mode works offline)

### Installation

```bash
git clone <repo-url>
cd Kasumi/frontend
npm install
```

### Development (Browser)

```bash
npm run dev
# ‚Üí http://localhost:5173
```

Starts in **mock mode** by default ‚Äî no Baserow connection required. Both Nexcel and WORDO shells are available immediately.

### Development (Electron)

```bash
ELECTRON=true npm run electron:dev
```

### Production Build

```bash
npm run build
# Output: dist/
```

### Electron Desktop Package (Windows)

```bash
ELECTRON=true npm run electron:build
# Output: dist-electron/ (.exe NSIS installer)
```

## Configuration

All runtime configuration is stored in `localStorage`. No `.env` file required.

| Key | Default | Description |
|-----|---------|-------------|
| `kasumi_use_mock` | `"true"` | `"true"` = MockAdapter; `"false"` = real Baserow |
| `kasumi_baserow_url` | `""` | Baserow server base URL |
| `kasumi_baserow_token` | `""` | Baserow API token |
| `kasumi_baserow_db_id` | `""` | Baserow database ID |
| `kasumi_access_mode` | `"analyst"` | `data-entry` / `analyst` / `admin` |
| `kasumi_active_shell` | `"nexcel"` | Last active shell (`nexcel` / `wordo`) |

Configure via the **Settings** button in the Ribbon, or set values directly in `localStorage`.
